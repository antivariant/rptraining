# InfernoSchool RP Dataset Builder

Локальное Streamlit-приложение для сбора RP-комикс-диалогов как датасета для LoRA fine-tuning character-conditioned RP модели.

## Что делает приложение

- пользователь по очереди играет разных персонажей в разных сессиях;
- LLM играет второго персонажа и выступает как sparring partner / provocateur;
- приложение сохраняет сырые сессии;
- post-processing позволяет редактировать сцену и диалог;
- экспорт формирует comic-level RP SFT, где модель учится отвечать за human-played character.

Экспорт обучает:

- character voice;
- humor;
- actions in square brackets;
- reactions to partner;
- scene progression;
- comic timing.

Экспорт не обучает:

- English lesson injection;
- grammar enforcement;
- vocabulary quotas;
- CEFR checks.

English lesson injection будет отдельным постобрабатывающим шагом после генерации RP-комикса.

## Архитектура данных

Приложение использует:

- character cards;
- location cards;
- scene description / initial state;
- requested frame count;
- raw session messages.

Карточки намеренно лёгкие и текстовые.

### Character card

- `name`
- `description`
- `likes`
- `dislikes`
- `catchphrases`
- `system_notes`

### Location card

- `name`
- `purpose`
- `description`
- `props`

## RP SFT export

Основной экспорт:

- `exports/sft_user_targets_rp.jsonl`

Формат:

- one valid raw session → one JSONL row;
- raw `assistant` messages → SFT `user`;
- raw `user` messages → SFT `assistant`;
- экспорт всегда заканчивается ролью `assistant`.

Контекст SFT включает:

- assistant character card;
- user character card;
- location card;
- scene description / initial state;
- requested frame count.

## Pipeline

1. Human plays target character in RP sessions.
2. LLM assistant acts as sparring partner.
3. App stores raw sessions.
4. App exports comic-level RP SFT.
5. RP model is fine-tuned with LoRA on human-written character replies.
6. Later, generated comics pass through separate English lesson postprocessing.
7. Later, comic images are generated by a separate SD-based image pipeline with character and location LoRAs.

## Setup

```bash
conda activate base
pip install -r requirements.txt
streamlit run app.py
```

## Endpoint defaults

Приложение работает с любым OpenAI-compatible endpoint.

По умолчанию:

- `dev` → `http://192.168.31.52:1234/v1`
- `prod` → `http://127.0.0.1:4000/v1`
- model → `qwen2.5-7b-instruct`

Имя модели должно совпадать с тем, что отдает выбранный endpoint.

## Langfuse

Поддерживается локальный Langfuse tracing.

Нужные переменные в `.env`:

```bash
LANGFUSE_SECRET_KEY=sk-lf-...
LANGFUSE_PUBLIC_KEY=pk-lf-...
LANGFUSE_BASE_URL=http://127.0.0.1:3000
```
